Goals #goals

Rework the order pipeline so a checkout completes in under two seconds at p95, without dropping orders during fulfillment-service restarts.

Non-goal: changing the public checkout API surface.

Latency budget #latency-budget updated by agent

Current p95 per stage, measured over the last 14 days. The budget allocates the two-second target across stages.

p95 latency by stage, ms. Amber line marks the 800 ms enqueue target; breaches shown in red.

Pipeline flow #pipeline-flow

Validation is split so schema failures reject fast, before inventory is consulted.

Retry policy #retry-policy updated by agent

Enqueue retries three times with exponential backoff and full jitter (base 200 ms, cap 2 s). After the third failure the order lands in the dead-letter queue and pages on-call; it is never silently dropped.
