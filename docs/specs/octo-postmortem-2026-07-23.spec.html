1 · The Herdr contract-fidelity seam: a fake that simulates beliefs

Strong
Files
elixir/test/symphony_elixir/herdr_transport_test.exs (fake-herdr fixture) · evals/role-workflows/implementer-herdr-delegation/* · .agents/skills/testing/SKILL.md · scripts/deploy-production-host (CD gate home)
Problem
The fake-herdr fixture is a hand-written simulator of the Adapter's upstream, not a recording of it. It synthesized responses the real binary cannot produce (e.g. agent_prompt_stalled under a 5000 ms wait) and drifted with each fix cycle to keep tests green. The interface of the HerdrTransport Adapter was therefore guessed, and reviewer + QA judged implementations against the same guess — a circular judge. All five 0.7.5 defects passed through this seam. The testing skill's External-contract fidelity doctrine forbids exactly this; the gap is enforcement, not instruction.
Solution
(a) A CD-gated real-binary contract suite: at deploy time (not CI — operator constraint: keep CI fast), run the transport's contract expectations against the pinned real herdr binary; fixture assertions carry provenance annotations pointing at the upstream behavior they record. (b) Role-behavior evals (EMB-1239, transcripts preserved) proving the implementer escalates rather than invents contract facts and the reviewer verifies against authoritative sources.
Benefits
Leverage: one contract suite pays back across every future transport change. Locality: contract drift is caught at the fixture, not five defects downstream in production panes. Tests stop validating beliefs and start validating the seam.

After

2 · A deep FailureEvidence module: fail loudly, carry everything

Strong
Files
elixir/lib/symphony_elixir/agent_runner.ex · orchestrator.ex (retry/backoff families, dispatch skip logic) · workspace.ex (hook_env nil-dropping) · role status endpoints and logs
Problem
Absorb-vs-report decisions are scattered and default to absorb: nil env values silently dropped (credential-less agents launched), deterministic failures retried forever, error payloads truncated below diagnostic use, dispatch skips visible only in rolling logs, agent process death surfaced as a misleading secondary symptom. The failure-classification boundary is a shallow module — many call sites each doing a sliver of classification and truncation — so diagnosis cost dominated the session (three defects were ultimately diagnosed by reading pane pixels).
Solution
EMB-1237 doctrine, realized as one deep module: every failure flows through a FailureEvidence interface that (1) types the failure at the earliest seam with no degraded continuation for active turns, (2) preserves the complete payload (untruncated provider output, command shape with secret names redacted, classification path), (3) justifies retryability — deterministic failures escalate after bounded attempts, (4) makes every skip/withhold queryable from the state endpoint.
Benefits
Locality: the "why" of any failure lives in one record, ending pane-forensics archaeology. Leverage: every role, every provider, every future defect inherits complete evidence. The deletion test passes: deleting today's scattered truncation sites concentrates complexity into the one module where it belongs.

3 · Generation-scoped runtime state with one cleanup owner

Strong
Files
bin/octo (launcher) · tracker/claim_lease.ex · herdr session state (~/.config/herdr/sessions, /tmp/octo-herdr-*) · role workspaces · dialyzer PLTs
Problem
State outlives process and cleanup has no owner. Every rollback/restart left landmines: stale leases blocked dispatch three times, persisted dead herdr sessions caused agent-name collisions, stale PLTs and workspace refs caused fresh-looking failures a cycle later. Cutovers flip binary+pin atomically but leave session state, leases, and workspace refs from the dead generation.
Solution
Every stack boot stamps a generation ID. Leases, herdr sessions, runtime roots, and workspace refs are tagged with it. The launcher owns a boot-time reconciler that sweeps foreign-generation state: supersede dead leases (folds in the parked lease-reconciliation work), purge dead sessions and roots, restore workspaces to expected branches. Cutover gains an explicit generation close-out step.
Benefits
Locality: cleanup logic in one owner instead of operator archaeology. Leverage: every future cutover, rollback, and crash-restart inherits a clean slate. Cutovers become actually atomic.

Before

gen N dies ─▶ leases, sessions, roots, refs
    └─▶ haunt gen N+1 (3 lease blocks, 1 name collision, 1 stale PLT)

After

boot(gen N+1) ─▶ reconciler sweeps tag≠N+1
    └─▶ clean slate, one owner, evidence logged

4 · Launch-projection secret hygiene

Strong · small
Files
elixir/lib/symphony_elixir/implementer_delegation/herdr_transport.ex (materialize_launch_projection) · leakage test matrix
Problem
Credential forwarding + self-contained projections compose into forwarded auth values being written into on-disk projection scripts (mode 0500, owner-only, session-lifetime — mitigated but real). The leakage test matrix covers argv, transcripts, and typed errors, not file persistence: a blind spot created by two individually-correct fixes composing.
Solution
Exclude the auth allowlist from embedded projection exports — panes already inherit the session server environment, so the env path carries credentials without disk persistence. Add file-persistence to the leakage test matrix so the composition can never regress.
Benefits
Locality: the secrecy invariant lives in one projection builder and one test. Closes the gap between the tested interface and the actual artifact surface.

5 · Console preflight: shift the validators left

Worth exploring
Files
scripts/pr-body-check · work_contract.py · symphony-post-turn-handoff-gate · (new) scripts/console-preflight
Problem
Validators are precise but their rules are discoverable only by failing them; roles absorb that across long turns, the console hits them in rapid succession (six PR-body CI round-trips in one night). The validator interface is effectively "submit and see" — shallow from the caller's perspective.
Solution
One console-preflight command bundling every submission gate (PR body template + heading order + foreign-key lint + contract TOML validation), runnable locally pre-push; each gate gains an --explain mode listing its rules. Documented in the octo-ops skill.
Benefits
Leverage: every future console (and role) submission pays zero discovery cost. Round-trips through CI collapse to one local command.

6 · Turn-context economics: routing carries authoritative context

Worth exploring
Files
Implementer delegation turn budget (turn_timeout_ms) · issue comment trails · routing macros · implementer-orchestrator profile
Problem
A big issue accumulated a comment trail whose reading time exceeded the one-hour turn budget: two full turns spent entirely on context acquisition, zero workspace changes, session context lost at each timeout — a doom loop. The turn interface assumes context fits in the budget; nothing enforces checkpoint discipline (commit + handoff before budget) or bounds the reading list.
Solution
Institutionalize what fixed it manually: (a) long-trail issues get an authoritative state note at routing time — verified facts + slice-sized remaining work — so agents never archaeology the trail; (b) checkpoint discipline in the orchestrator profile: commit and forward-handoff well inside the budget, partial progress is a valid turn; (c) optionally scale turn budget with effort tier.
Benefits
Locality: progress persists in the workspace every turn instead of evaporating at timeout. Leverage: every future mega-issue inherits the pattern; the two lost hours become the last ones.

7 · The pane shell is a hostile boundary — codify it

Worth exploring
Files
docs/specs/domains/agent-runtime.md · herdr_transport.ex (all pane-crossing paths)
Problem
Three of five defects were the same lesson relearned: anything crossing the pane shell as typed input or ambient environment is unreliable (paste chunking, quote mangling, profile PATH resets). Each fix rediscovered the boundary's hostility independently.
Solution
A one-paragraph spec invariant: nothing crosses the pane shell except short, quote-trivial references to self-contained artifacts; no correctness may depend on pane environment ordering or paste integrity. Every future pane-touching change is reviewed against it. Largely already implemented — this codifies it so it cannot be un-learned.
Benefits
Converts three hard-won production lessons into one reviewable sentence. Cheap insurance with permanent leverage.

Adversarial review — GPT-5.6 Sol (high) · debate · final consensus

Sol's headline attack: the ranking was backwards — the fake didn't create four of five defects, it blinded detection; the root mistake is treating the interactive pane as a process-execution protocol. Conceded, with the feasibility bound that Herdr's pane model is the product. Consensus after one debate round:

  • A · New top invariant (was card 7, elevated): the pane PTY is transport of last resort; every crossing is a fixed tiny bootstrap reference; all control flow uses the typed Herdr API; structured launch/prompt injection filed upstream.
  • B · Card 1 amended: fast hermetic real-binary smoke + fake-vs-real differential checks in CI; full shell/locale/size/race matrix at nightly/CD. Tiering, not exclusion.
  • C · Role evals positioned: process-layer controls on agent discipline and fleet regressions (EMB-1238/1239); executable schema/differential checks remain the merge gates.
  • D · Card 3 amended, topology-bound: durable controller epochs + desired/observed reconciliation with adopt/resume/close/quarantine/GC verbs + tombstoned idempotent cleanup. Distributed fencing deferred until multi-host/rolling deployment exists.
  • E · Card 4 amended: per-launch provider-process env injection preferred; session-server inheritance is documented debt only where Herdr offers nothing narrower — no file persistence, rotation notes, upstream structured-injection request.
  • F · Card 2 amended: split into failure envelope / bounded evidence store (excerpts inline, blobs by hashed reference) / redaction / retry controller / dispatch state; explicit acceptance-state field (not_sent | sent_unknown | accepted | completed); "untruncated everywhere" rejected as unsafe.
  • G · Card 5 amended: preflight must execute the exact CI validator code and versions — a frontend, never a parallel implementation.
  • H · Card 6 amended: system-generated versioned task dossier with verifiable source links; typed checkpoint artifacts (commit, patch, repro script, test result, evidenced hypothesis, decision log) — transcript prose is never completion evidence.

Revised top recommendation: the pane-transport invariant (A) leads as the standing architectural rule; Card 1-as-amended (B) and Card 2-as-amended (F) follow as the paired detection + evidence work.
